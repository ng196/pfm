# Worldbuilding Bible

Build the world *as a tool for story*, not as an encyclopedia.

## Default sections to create or update

If the project structure exists, write/update:
- `bible/world.md`
- `bible/magic-system.md`

Include:
- 3–5 “anchor images” (memorable visual motifs)
- Factions (goals, resources, taboo, leverage)
- A timeline of 5–10 “history pressure points” that still affect the present
- Magic rules with explicit costs and failure modes

## Constraints

- Magic must have: `rules`, `costs`, `limits`, `failure modes`, and `who can use it`.
- Culture is shown through: food, greetings, funerals, laws, and what people fear.
- Avoid dumping lore; prioritize what the next 3 scenes need.

## Deliverable style

- Use short headings and bullets.
- Make it easy to reference while drafting: define terms consistently.
